🛠️ Chapter 7: Exception Handling and Debugging in Python

🚨 Why This Matters?

“To err is human… but to handle it smartly is Pythonic!” 🧠🐍

Even great programs crash if you don’t handle unexpected situations like:

  • • File not found
  • • User entering wrong input
  • • Dividing by zero

⚡ What is an Exception?

An exception is a runtime error that interrupts your program's normal flow.

print(10 / 0)       # ZeroDivisionError
int("abc")          # ValueError
open("nope.txt")    # FileNotFoundError
😅 You: “What’s your age?”
User: “Two cats and a banana.” 🐱🍌
Your code: 💀 dies with ValueError

🛡️ Python to the Rescue: try...except

try:
    x = int(input("Enter a number: "))
    print(100 / x)
except ZeroDivisionError:
    print("Oops! Can't divide by 0.")
except ValueError:
    print("That was not a number!")

🧪 Output:

Enter a number: 0
Oops! Can't divide by 0.

🔁 Multiple except Blocks

try:
    num = int(input("Enter number: "))
    result = 10 / num
except ValueError:
    print("Only numbers allowed!")
except ZeroDivisionError:
    print("Don't divide by 0, please.")

🧼 finally: Always Executes

try:
    f = open("file.txt", "r")
    data = f.read()
except FileNotFoundError:
    print("File missing!")
finally:
    print("Closing program.")

Even if there’s an error, finally block runs — like brushing your teeth before bed 😄

🛠️ else: Runs only if no exception

try:
    x = 5
    y = 2
    result = x / y
except:
    print("Something went wrong.")
else:
    print("Everything worked fine!")

🔧 Real-World Scenario: Login System

try:
    username = input("Username: ")
    if username == "":
        raise ValueError("Username can't be blank")
    print("Welcome,", username)
except ValueError as e:
    print("Error:", e)

🐞 Part 2: Debugging Tips 🕵️

🔎 What is Debugging?

Debugging means finding and fixing bugs (errors) in your code.

🧰 Common Tools for Debugging

  • 1. print() debugging – Old but gold 😎
  • 2. Using IDE Debugger – Step-by-step execution
  • 3. pdb module – Built-in debugger

✅ Example: Using print() to find bugs

def calculate_area(length, width):
    print("Length:", length)
    print("Width:", width)
    return length * width

print("Area:", calculate_area(5, 2))

🐍 Pro Tip: Use assert for quick checks

def divide(a, b):
    assert b != 0, "b should not be zero!"
    return a / b
