🧱 Chapter 8: Object-Oriented Programming (OOP) in Python

🤔 Why OOP?

“Programming without OOP is like trying to build a house without bricks.”

  • More organized
  • Reusable (Write once, use many times)
  • Scalable (Great for large applications)
  • Based on real-world objects

🧱 Step 1: Creating a Class

class Student:
    def __init__(self, name, grade):
        self.name = name
        self.grade = grade

    def show(self):
        print(f"{self.name} scored {self.grade}")

🔧 Step 2: Creating an Object

s1 = Student("Aman", 95)
s1.show()  # Output: Aman scored 95

🛠️ __init__() Method

  • Called automatically when object is created
  • Used to initialize values (like a constructor)
def __init__(self, name, grade):
    self.name = name
    self.grade = grade

🧑‍👩‍👦 Inheritance

Let’s reuse code using inheritance!

class Person:
    def __init__(self, name):
        self.name = name

    def say_hello(self):
        print("Hi, I’m", self.name)

class Student(Person):  # Student inherits Person
    def study(self):
        print(self.name, "is studying.")

s2 = Student("Riya")
s2.say_hello()
s2.study()

🎭 Polymorphism Example

class Animal:
    def speak(self):
        print("Animal speaks")

class Dog(Animal):
    def speak(self):
        print("Dog barks")

a = Animal()
d = Dog()

a.speak()  # Animal speaks
d.speak()  # Dog barks

🎒 Encapsulation

class BankAccount:
    def __init__(self, balance):
        self.__balance = balance  # Private attribute

    def show_balance(self):
        print("Balance:", self.__balance)

acc = BankAccount(1000)
acc.show_balance()
# acc.__balance  ❌ Error (private)

🔍 Abstraction

Hide complex parts, only show essentials!

from abc import ABC, abstractmethod

class Vehicle(ABC):
    @abstractmethod
    def start(self):
        pass

class Bike(Vehicle):
    def start(self):
        print("Bike started!")

b = Bike()
b.start()
